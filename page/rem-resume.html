建议汇总

整体

一般遵循以下原则：

1、诚信，不造假。
2、满怀激情：我特别想入职贵单位。
3、只写和求职岗位相关的+重要的。不写不相关的或不重要的。
4、重要的往前放。
5、呈现最美的80%人生，不主动暴短。
6、排版优美、清晰。
7、正确使用英文大小写。
8、正确使用中文标点符号。
9、怎么好怎么写，凡是能呈现高水平的。
10、自信满满：岗位三千，我只取一个就之！我是最棒的！

一般应呈现的信息有：

✳️ 基本信息
✳️ 教育背景
✳️ 专业技能
✳️ 项目经历
✳️ 实习经历
✳️ 校园经历
✳️ 自我评价

基本信息

建议呈现：

❇️ 名字
❇️ 电话
❇️ 邮箱
❇️ 班委/团委（如有）
❇️ 中共党员/预备党员（如是）
❇️ 照片
   - 微笑的正装照/形象照，能体现青春活力、良好的精神面貌的那种。
   - 不是严肃的证件照，不是手机自拍的杂乱背景照，不是卡通头像。
❇️ 求职岗位。写上可能有利于投递某些岗位，不一概而论。

可以不写：

🆖 年龄
🆖 学校。因为在 [教育背景] 段落中有了。
🆖 学历。因为在 [教育背景] 段落中有了。
🆖 身高
🆖 居住地
🆖 籍贯
🆖 民族
🆖 期望薪资
🆖 期望工作城市

教育背景

建议呈现：

❇️ 学校
   - 入学毕业时间都写年月，不写“xx年xx月~至今”。
   - 可列明 211双一流（如果用人单位可能不知道）。
❇️ 课程
   - 写主要的、和求职岗位相关的几门即可，控制在一行。
   - 如果某些核心/主要课程满绩，可在课程名称后备注下。体现成绩优秀。
❇️ GPA
   - 如GPA在 3.2 左右可写，如：GPA 3.3/4.0(专业排名前15%)。3.2只是举例。
   - GPA不高则不写。但要想好面试时怎么说。
❇️ 奖项荣誉
   - 含金量高的写上，其他可不写。否则显得不高级。
   - 简洁、明了。比如“ACM ICPC（国家大赛）三等奖（2023）”，可能比 “ACM国际大学生程序设计竞赛 Java 组中国区三等奖” 好一些。比如“蓝桥杯（工信部主办）二等奖（2021）”，可能比“2021届 蓝桥杯 Java组 A组 省赛二等奖”好一些。
[技能] 列出和求职岗位相关的+重要的，不相关或不重要的不写。
[技能] 不谦虚，不过分夸大。能力程度:精通 > 掌握(即熟练掌握)> 熟悉 > 了解。 v [技能] 可单列一段“专业技能”放在教育背景下面。

[咨询] 我想请教下如果投递像银行或者学而思这类比较类似于管培生或者强调的内容非技术类的项目实习的简历该怎么改呢

[答复] 相关建议如下，仅供参考。

一、银行招聘要求

假定银行招聘要求如下：

职位描述
金融科技储备生分为开发应用、数据分析、产品经理等方向，主要从事业务条线数据规划管理、数据挖掘分析和应用、系统软件开发和优化、金融产品研发和优化、客户需求挖掘、金融服务方案设计等工作。以培养分行数字化专业型人才为目标，聚焦银行业金融科技核心领域，通过专项业务培训、轮岗工作实践、重大项目历练、导师带教指导，提升业务管理技能和综合素质，通过基层锻炼、专业岗位实践、项目部门轮岗等方式，培养金融科技与业务相融合的复合型人才。

职位要求
(1)大学本科及以上学历和学位（教育部认可的国内统招或海外院校）的应届毕业生；
(2)IT技术类、数理分析类、电子通信类、经管财会类、法律语言类等专业优先；
(3)大学英语四级及以上水平，主课成绩优秀，掌握一定金融、经济或银行业基础知识；
(4)熟悉并掌握计算机软硬件基础知识，熟悉WINDOWS、LINUX操作系统管理、网络相关知识；
(5)品学兼优，政治素质良好，具有较强的学习能力、沟通能力和团队协作精神，具备较好的仪表气质和积极向上的精神面貌，适应能力较强并能承受一定的工作压力；
(6)身体健康，无任何违法违规违纪行为，符合 xx 银行履职回避及招录回避规定。

并假定你希望从事开发应用。从招聘描述看，开发应用、数据分析、产品经理的主要职责如下：

  • 开发应用：主要从事业务条线系统软件开发和优化、金融产品研发和优化等工作。
  • 数据分析：主要从事业务条线数据规划管理、数据挖掘分析和应用等工作。
  • 产品经理：主要从事业务条线客户需求挖掘、金融服务方案设计等工作。

已有，简历中说通过CET-6。
- 如果成绩比较好，还可以列下成绩，并注明前x%之类的。
- 建议放在 [职业技能] 段落的最后。

暂无。

2.4 熟悉并掌握计算机软硬件基础知识，熟悉WINDOWS、LINUX操作系统管理、网络相关知识
有一些。
建议在 [职业技能] 段落再多写一些。上过相关课程，有过很多实践，应该可以写出来。建议放在前面些。

- 如果有比较好的 [校园经历] 或 [社会实践]，可以体现出上述要求的，建议补充 [校园经历] 段落，放在倒数第2段（最后1段是 [自我评价]）

2.6 具有较强的学习能力、沟通能力和团队协作精神，……，适应能力较强并能承受一定的工作压力

2.7 具备较好的仪表气质和积极向上的精神面貌
基本具备。
当前简历照片可以考虑换一张。问问女生哪边有拍照的，拍形象照（不是严肃的证件照）。如果不知道哪里可以拍，可联系我。

三、其他修改点的建议

3.1 基本信息

  • 年龄：可写可不写。
  • 籍贯：可写可不写。除非有助于获取 offer。比如南通银行希望招南通籍毕业生，那可以写上籍贯（不写也OK）。
  • 求职意向
    • [银行] 删除
    • [Java] 维持现状
  • 如是党员/预备党员、班干部、团干部、学生会干部等，可呈现在这里。（新增如有）
  • 照片：参考 2.7

3.2 教育背景

  • 课程（新增）
    • [银行] 参考2.2
    • [Java] 列几门和Java岗相关的课程，一行为宜。
  • GPA。排名不是特别靠前，建议删除。面试问起GPA，考虑怎么答。不造假，不主动暴短。还是要体现学习能力强。比如打比赛的某同学，核心课程差不多满绩，副科只是通过，因为时间花在刷题打比赛了，因此GPA不是很高。
  • 荣誉奖项（新增）
    • 第十五届蓝桥杯 A 组省级三等奖 --> 蓝桥杯（工信部主办）三等奖（2023）。2023改成正确的获奖年份。
    • 叁等综合奖学金 --> 三等奖学金（2023）。2023改成正确的获奖年份。
  • 软著/专利/论文（新增如有）
    • [银行] 如有则新增呈现。
    • [Java] 可写可不写，都行。

3.3 职业技能

  • 职业技能 --> 专业技能？
  • [银行] 参考 2.1、2.3、2.4
  • [银行] 现有描述浓缩后，部分内容也可放进来。比如：掌握 Java，可能不需要进一步写 “包括常用集合类、IO流、多线程等概念。”
  • [Java] 去掉基础、常用、常见、概念等字样，显得不高级。核心、主流等，这些词就很好。
  • [Java] 按岗位要求的重要程度 + 自身技能程度 综合排序，将最希望被看到的技能放前面些。
  • 一行描述一种技能，或几种相关的技能。比如 MySQL、Oracle可以放在一行写。Redis是否要单独写，还是和MySQL写在一行？不要把 Java 和 Linux 写在一行，除非按某种更大的分类逻辑，TA 们是相关的技能。
  • 做成列表，每行前面加个 .，可能更清晰直观。就如同本行文字前面有个. 一样。
  • 技能程度一般是： 精通 > 掌握（熟练掌握） > 熟悉 > 了解。不要谦虚，也不要夸大。你只是本科应届生。可以网上搜搜技能程度，供参考，不一定非常严格对应，大致符合常识理解即可。
  • 因此，“了解并使用过”改成“熟悉”？“了解并掌握”改成“掌握”？还有类似地方，不一一列举。
  • 岗位对某些技能要求是“掌握”，而当前自身只是“了解”，则建议先写掌握，然后在很短时间内提升到熟悉，并尽快达到掌握级别。要求掌握但只是了解，可能过不了简历关。以学辅投，以投促学。

3.4 项目经历

  • 建议按 STAR 规则描述，可自行搜 STAR 是什么。大致是：是什么（最好有为什么要做该项目，一般是为了解决什么问题），你负责什么，通过什么手段解决了什么，最终的结果/效果如何（必须是很棒！）。当前的描述，看上去只是介绍了某个项目，似乎和你一点关系都没有。
  • 通过简要描述，让能够看到简历的人觉得你似乎很厉害。否则HR筛选简历就过不了。真正识货的面试官会提出相关技术问题的。当前描述有点散、多。
  • 项目概述。写了3-4行，有点多。不是普及这个项目到底是什么，每个字都是为了反复说“我很厉害”服务的。按 STAR 规则重新打散组织。
  • 业务功能。写了4行，有点多。对于拼车软件，可能最核心的功能是提高成交率。按 STAR 规则重新打散组织。
  • 技术栈。可以简要列几个，融到 T 或 A 中。比如，我选用了 xx、xx、xx 实现/解决了 xx。
  • [银行] 把当前描述进一步浓缩后呈现，让人觉得你做了很棒的项目，你很厉害，即可。不必太多展开技术细节。

3.5 实习经历

  • 如有和岗位相关的实习经历，也可以写上。也按 STAR 类似方式写。
  • 可以放在项目经历之前或之后。

3.6 校园经历

  • 参考 2.5。
  • 能体现软能力的、其他优秀品质的，都可以写。
  • [Java] 也是要写的。只有专业技能 + 软能力 + 其他品质都符合，才能拿到 offer。专业技能很棒，其他不符合，则拿不到 offer。

3.7 自我评价

  • 参考 2.6。
  • [Java] 也是要写的。参考 3.6。

写在最后

  • 每个字都在说我很厉害。要惜墨如金。只要觉得能帮助获得 offer的，都可以写。不相关的不要写。
  • 关注细节。标点符号是否正确，英文大小写是否正确，有没有错别字，语句是否通顺，意思表达是否清晰，是否容易阅读（HR可能一天要看数十上百份简历），重点内容是否需要加粗，……

xx同学好！ 针对 24-09-30 版本的简历，有如下建议，仅供参考。预祝同学早日找到心仪岗位！

1、缺少内容，要补齐。 本科应届生求职简历一般按顺序有以下主要内容：

  • 基本信息。名字，联系方式（手机，邮箱），等等。（已有）
  • 教育经历。学校，专业，起止时间，主要相关课程，GPA（如排名相对靠前），荣誉奖项，等。（已有）
  • 专业技能。针对拟求职岗位，所具备的专业技能的描述。比如熟悉C/C++，……。（基本未涉及）
  • 项目经历。做过哪些项目，尤其是和拟求职岗位相关的。项目可以来自于：课程设计/大作业，毕业设计，各种竞赛，网上开源项目，其他来源，等等。按 STAR 法则描述项目内容。（基本未涉及）
  • 实习经历。（尚无）
  • 校园经历。社会实践，等等。（基本未涉及）
  • 自我评价。3到5行，每行描述一个或相关软能力（沟通、协调、组织等等）。（尚无）
  • 其他。其他有助于获得 offer 的内容。

2、整体内容要调优

  • 按上述第 1 条建议补齐内容。
  • 不足 1 页时，通过补充内容、优化版面等方式，扩充到 1 页。（当前有效内容约只有半页）
  • 原则上简历不要超过 1 页。可通过优化内容和优化版面的方式控制在 1 页内。
  • 要正确使用中文、英文的标点符号。不一一指出了。
  • 要正确使用英文的大小写。比如 mysql 应该是 MySQL， python 应该是 Python，等等。
  • 内容要准确。比如，“相关课程：”后面紧跟着“相关课程：”，比如学院的名称其实不是“计算机与人工智能学院”，……。
  • 只呈现能有助于获取 offer 的内容，要惜墨如金。可能爱读书、喜欢健身，不一定有助于获取你喜欢的 Python 或 Java 开发岗。

3、整体版面要调优

  • 缩进似乎没有规律。要调成一致。
  • 行内有多个多余空格。
  • 教育经历等标题，超出了后面的灰色背景。

4、基本信息。有如下建议，供参考。

  • 页面顶部空白太多，要减少。
  • 电话号码 --> 手机
  • 邮箱。写了 2 个，建议删除 1 个，仅保留 1 个。
  • 当前所在城市。删除。
  • 出生日期 + 后面的xx年xx月xx日。删除。
  • 在校生。删除。
  • 期望薪资。删除。
  • 班委团委。如做过，可写在这里。
  • 民族。如觉得有助于获取offer，则可以写上。
  • 政治面貌。如是中共党员或预备党员，则可以写上。

5、教育经历

  • 学校名称。后面有2个浅蓝色框框，请修改。
  • 可以考虑在学校名称后面增加：（211；双一流）
  • 学院名称。可删除，或修改为正确的名称。
  • GPA。如专业排名在 20% 以内，则可以写上，比如 GPA： 3.3/4.0（专业排名前17%）。如不是则不列了，不主动暴短。同时准备好面试时如何回答GPA相关的问题，不要造成学习能力不行的印象。
  • 荣誉奖项。简短些，比如修改为：xx训练营证书（2024）。普公英可写可不写，如要写则简短些，比如：普公英三等奖（2023）。

6、专业技能（当前基本未涉及）

  • 列出拟投岗位所需技能的掌握程度。比如想做 Python 或 C++ 开发岗，岗位要求12345专业技能，你正好都具备，用人单位HR初筛看到这个同学貌似都符合，就可能进入下一阶段。当前缺。
  • 精通 > 掌握（熟练掌握） > 熟悉 > 了解。不要谦虚，也不要夸大。
  • 岗位相关性 + 技能掌握程度要综合考虑，重要的放前面。比如某岗位要求熟练掌握C++，则可能要把C++放第一条，而不是：第 1 条写熟悉 Office软件，第 2 条写熟悉 SQL，……，第 10 条才写熟练掌握 C++。
  • 作为计算机专业的同学，不管应聘什么岗位，尤其是编码岗位，建议把“熟练掌握 office 软件”放在专业技能段落的靠后位置。如果放在靠前位置，可能会被误解：编程能力不佳。

7、项目经历

  • 相关经历 --> 项目经历
  • 建议采用 STAR 法则描述项目。比如，是什么/为什么要做这个项目，用什么方法，解决什么问题，取得什么成果。一句话，简洁明了。紧紧围绕说自己有多么厉害，而不要对项目做科普。比如：根据业务需求对数据应用特征工程与重要性分析，搭建机器学习模型， 应用数据增强和成本敏感学习解决不平衡数据集，准确率达到 83%。当前的描述略显不足：做了xx微信小程序，做了xx管理系统。
  • 和所投岗位强相关的项目，放前面一些，也可以多写几句话。不强相关的项目，可以放后面一些，也可以少写几句话。
  • 建议再补1-2个项目，和 Java / Python 比较相关的。项目来源，可参考“1、缺少内容，要补齐”。当前项目数量够了，但和 Java / Python 相关的不多。

8、实习经历/校园经历等

  • 如有相关性较大的实习经历，可以考虑实习经历单独成段落并适当放前面些，比如放在专业技能和项目经历之间，甚至更前面些。有的招聘描述有“具有xx实习经历的同学优先考虑”，恰好你有，则考虑按此处理。
  • 实习经历也按 STAR 法则描述。
  • 实习经历和拟投岗位强相关的，可多写几句，并放实习经历段落的前面些。反之，可少写几句或不写，并放后面些。
  • 实习经历和校园经历，可混在一起写。
  • 校园经历也尽量按 STAR 法则写。不要记流水账，比如我做了什么什么。不要空泛，比如提升个人能力（最好是个人的什么能力）。牢记每个字都是为了说自己有多么好、多么厉害、多么适合所投岗位。
  • 觉得有助于获取 offer 的内容，都可以写上。
  • 该版简历中的助理经历，建议移到校园经历中。
  • “提高了编程和团队合作能力……”，是针对哪段经历的？从上下文看似乎针对助理的经历，但从逻辑上看助理经历应该无法提升编程能力。

9、自我评价/自我小结

  • 需要增加“自我评价”（或自我小结）段落，放在简历最后。描述自己的软能力（比如沟通、组织、协调、……）。
  • 3-5行，每行描述一个软能力，或几个相关的软能力。最好能简明扼要重复项目经历或实习经历或校园经历，予以佐证，而不是自说自话、自我标榜。
  • 3-5个就差不多了。仅写1个的话，觉得你不大重视求职，或者软能力很少。写10个的话，觉得有夸大成份，用人单位也来不及看，有点喧宾夺主。
  • 重点描述几个所投岗位所需要的软能力。
  • 不写兴趣爱好，除非能将爱好成为某个软能力的佐证。比如学习能力强，我每年能阅读超过 xx 本 xx 类型的书籍（只是打个比方，这个例子不一定成立）。


写在最后：找什么岗位

自己能够获得的。而不是什么热门就投什么岗位。
不一定非要编码。各行各业的各种岗位都需要计算机人才，不一定非要投编码岗位。
找一些能力交叉岗位。喜欢销售的同学可以投招计算机专业的那种销售。懂一些前台的 Java 同学，可以投“了解前台的优先”那些岗位。
