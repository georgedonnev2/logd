整体建议

有如下整体建议，供参考。

▪️ 每个字都是为了“我很棒！我很适合该岗位！”
// 惜墨如金。
// 通常写满 1 页 A4 纸。
// 不写不相关的，或不重要的。比如通常不写有驾照。

▪️ 诚信，不造假
// 一旦发现造假，则无望获得 offer。

▪️ 呈现最美的80%，不主动暴短
// 比如只呈现若干核心课程满绩，但GPA 3.0 就不写在简历上了。
// 比如不要主动呈现：GPA 2.0，专业排名前 90%。但要想好用人单位问GPA时怎么回答。

▪️ 重要的往前放
// 排在前面，排在左边。符合从上到下、从左到右的阅读习惯。
// 比如国家一等奖学金，就放在 [荣誉奖项] 那行的最左边。
// 比如有在大厂相关的实习经历，就把 [实习经历] 放到 [项目经历] 前面。

▪️ 排版优美、逻辑清晰
// 我技术好，不整这些虚的。要整这些虚的！一堆985、海归，技术好，简历又精致。
// 简历写得逻辑清晰、优美精致，也是一项被用人单位看重的能力。
// 为了每天筛选几百份简历的 HR 小姐姐的眼睛考虑，也是要优化排版的。
// 用好看的字体，大小适中。既显得高级，又方便阅读。
// 类似内容要统一字体。比如项目1描述用宋体、18px，项目2描述用微软雅黑、24px，2个项目不统一。
// 合适的行间距，以方便阅读。不要行与行紧紧挤在一起。
// 合适的字间距（如可以调的话），以方便阅读。
// 可加粗和（或）不同颜色，突出最想用人单位看到的信息。重点不要过多，过多就没有重点了。
// 排版四大原则：对比、重复、对齐、亲密。

▪️ 正确使用英文大小写
// 我技术好，不整这些虚的。要整这些虚的！科研和工作都要严谨！
// 我是 MySQL。我不是 MYSQL、mysql、mySQL。
// 我是 Python。我不是 python、PYTHON。

▪️ 正确使用中文标点符号
// 该逗号就逗号，该顿号就顿号。不要一逗到底。
// 更多详情请参考 国标GB/T 15834-2011 标点符号用法🔗

▪️ 满怀激情：我特别想入职贵单位
// 发挥了主动能动性，就可以想到更多好的方法措施。
// 做一次销售，把自己“卖”个好价钱，就会想到更多好的方法措施。很有意思的！

▪️ 自信满满：岗位三千，我只取一个就之！我是最棒的！
// 截至2024年6月20日，全国高等学校共计3117所，其中：普通高等学校2868所，含本科学校1308所、高职（专科）学校1560所；成人高等学校249所。本名单未包含港澳台地区高等学校。更多详情可参考：全国高等学校名单🔗。咱们还是有竞争力的！
// 第二轮“双一流”建设高校及建设学科名单中，咱们是 140 多所高校之一，还是有竞争力的！更多详情可参考：第二轮“双一流”建设高校及 建设学科名单🔗。
// 参考某大模型估计和应届毕业生数量，每年大致有千万级岗位提供给应届毕业生。IT/互联网/游戏行业招计算机类岗位约 200-300万，加上其他招聘计算机类毕业生的行业（如电子/通信/半导体行业、金融行业等），则更多。
// 某大厂招销售岗，来学校举办了销售岗宣讲会。（某大厂销售岗较宽，不仅仅是传统意义上的销售卖东西，详见该大厂投递建议）
// 咨询某大厂的某产品线HR，得到答复：该产品线研发岗也招咱们学校的。
// 手握数十个 offer 的牛娃，也只能入职一个单位。牛娃最终选择后空出来的岗位，也是机会！

▪️ 怎么好怎么写
// 只要让用人单位觉得：你很厉害！你很适合所投岗位！


简历内容

应届毕业生的求职简历，一般应按如下顺序呈现如下相关信息。当然，建议的顺序也不是绝对的，但一般应迎合用人单位一眼扫过去就能看到想知道的信息。或者你希望用人单位优先看到的信息，比如将 [实习经历] 放在 [项目经历] 前面，因为正好在某大厂实习过。

▪️ 基本信息
// 照片（积极向上、青春阳光的形象照）。
// 姓名，手机，邮箱，等。
// 其他有助于获取 offer 的信息。

▪️ 教育背景
// 学校，专业，入学毕业时间。
// GPA（排名靠前则呈现）。
// 核心课程，或和求助岗位强相关课程。一行左右即可。
// 荣誉奖项，等。
// 其他有助于获取 offer 的信息。

▪️ 专业技能
// 参考岗位要求，描述你具备的相关技能。
// 以及技能的程度（精通 > 掌握 > 熟悉 > 了解）。
// 其他有助于获取 offer 的专业技能（通常不是吹拉弹唱）。

▪️ 项目经历
// 应届毕业生的项目来源很多，不仅仅是和企业做项目。
// 参考网上教程中的项目样例，从头到尾敲一遍代码，也是项目。
// 通常可参考 STAR 法则叙述。能呈现用人单位想知道的信息，同时说明你很强。
// 项目经历描述的目的只有一个：我很棒！我很适合该岗位！

▪️ 实习经历
// 有相关性较强的实习经历，是最好的。
// 往届没有实习经历的校友，也找到了心仪企业的心仪岗位。
// 通常可参考 STAR 法则叙述。能呈现用人单位想知道的信息，同时说明你很强。
// 实习经历描述的目的只有一个：我很棒！我很适合该岗位！

▪️ 校园经历
// 其他经历（不是项目经历，不是实习经历）写在这里。
// 有助于获得 offer 的经历，都可以写。
// 用于佐证你的优秀品质 / 软能力。
// 也可参考 STAR 法则叙述，相比项目经历/实习经历，要更精炼些。
// 尽量写。用人单位关注校园经历的。

▪️ 自我评价
// 类似总结。偏软能力（沟通、协作等），一般不重复专业技能。
// 3~4行。每行写 1 个软能力。
// 写少了，觉得你不重视；写多了，用人单位来不及看。


通常不写：

▫️ 兴趣爱好 （通常不写在简历中）
// 通常无法有助于获得 offer。所以不写。

简历上出现一些不大相关的信息，通常是简历模板导致的。一般在招聘网站上可以设置，虽然网站有一些必填信息（比如期望薪酬等），但可以不显示在电子版简历上。


基本信息

建议呈现：

❇️ 名字
❇️ 电话
❇️ 邮箱
❇️ 班委/团委（如有）
❇️ 中共党员/预备党员（如是）
❇️ 照片
▫️ 微笑的正装照/形象照，能体现青春活力、良好的精神面貌的那种。
▫️ 不是严肃的证件照，不是手机自拍的杂乱背景照，不是卡通头像。
❇️ 求职岗位。写上可能有利于投递某些岗位，不一概而论。

可以不写：

🆖 年龄
🆖 学校。因为在 [教育背景] 段落中有了。
🆖 学历。因为在 [教育背景] 段落中有了。
🆖 身高
🆖 居住地
🆖 籍贯
🆖 民族
🆖 期望薪资
🆖 期望工作城市


教育背景

建议呈现：

❇️ 学校
▫️ 入学毕业时间都写年月，不写“xx年xx月~至今”。
▫️ 可列明 211双一流（如果用人单位可能不知道）。
❇️ 课程
▫️ 写主要的、和求职岗位相关的几门即可，控制在一行。
▫️ 如果某些核心/主要课程满绩，可在课程名称后备注下。体现成绩优秀。
❇️ GPA
▫️ 如GPA在 3.2 及以上可写，如：GPA 3.2/4.0(专业排名前15%)。3.2只是举例。
▫️ GPA不高则不写。但要想好面试时怎么说。
❇️ 奖项荣誉
▫️ 含金量高的写上，其他可不写。否则显得不高级。
▫️ 简洁、明了。比如“ACM ICPC（国家大赛）三等奖（2023）”，可能比 “ACM国际大学生程序设计竞赛 Java 组中国区三等奖”好一些。
▫️ 简洁、明了。比如“蓝桥杯（工信部主办）二等奖（2021）”，可能比“2021届 蓝桥杯 Java组 A组 省赛二等奖”好一些。
▫️ 简洁、明了。比如 “校三等奖学金（2020、2021、2022）”，可能比 “xx大学 2020年综合杰出优秀奖学金A组三等奖金”、“xx 2021年 xx”、“xx 2022年 xx”要好些。
❇️ 论文专利
▫️ 含金量高的写上，其他可不写。否则显得不高级。
▫️ 建议也是简洁明了。让人觉得你还厉害。
❇️ 其他信息。 其他可帮助获得 offer 的信息，也可以呈现。


专业技能

▪️ 列出和求职岗位相关的技能。 不相关或不重要的，建议不写。
// 目的是让用人单位觉得你具备岗位所需技能，要不做个笔试面试呗。

▪️ 呈现合理的技能程度。 一般是：精通 > 掌握(即熟练掌握) > 熟悉 > 了解。
// 慎写精通，因为需要时间积累。
// 不夸大，更不能谦虚。
// 确保达到岗位要求的程度。比如岗位要求掌握，自身只是熟悉，可以先写掌握，并尽快学习提升到掌握（至少是半步掌握）。否则可能过不了简历筛选关。

▪️ 一行写一个（或多个相关）技能
// 比如一行可以写：“掌握 C/C++/Python，……”，都是说编程语言。（不是说一定要把几门编程语言写一行）
// 不要一行写：掌握 C/C++，熟悉 MySQL，了解 TCP/IP。

▪️ 重要的 + 相关的技能，放前面
// 英语四六级 & 普通话，可写一行，都是语言。但要靠后放，作为最后几项技能之一。除非岗位对英语有特别要求，则可以靠前放。
// Office 可写，今后工作中也常用。也要靠后放。如果放在前面，会被误解计算机技能不足的。


项目经历

▪️ 来源较多，都可以作为项目写。 来自课程设计、竞赛项目、自己做的项目、毕设，等。

▪️ 说清楚是什么 / 做了什么 / 效果如何等，突出自己很厉害。 可参考 STAR 法则描述。
// 不要记流水账。每一个字都要说自己很厉害。
// 不要做项目科普，书面呈现很多细节。每一个字都要说自己很厉害。
// 项目很有价值，有很多难点，我解决了很多，效果很棒。就看上去很厉害！

▪️ 符合信息安全和商业保密。 未经授权披露的信息，不要写。
// 比如在 A 公司针对 B 客户做了 C 项目，而用人单位 D 公司和 A 公司是商业竞争关系，你就泄密了。

▪️ 项目标题那行中，突出项目的公司、或项目名、或自己的角色。
// 公司很有名，就突出公司。当然，要符合信息安全和商业机密，至少可以写某某世界500强企业。
// 项目看上去很厉害，就突出项目。比如基于xx的人工智能xx项目。
// 如果角色厉害，就突出角色。比如总体负责，比如核心成员，等。
// 项目的起止时间，通常写在项目标题那行的行末，而不是行首。


实习经历

和项目经历类似的写法和要求。按投递企业和岗位不同，可考虑放在 [项目经历] 之前，或之后。


校园经历

校园经历可描述在校期间参加的社团活动、社会实践等。即不属于项目经历的、不属于实习经历的，但有助于获取 offer 的经历，都可以放在这里。对获取 offer 没有帮助的经历，就不呈现在简历上，因为版面有限。

校园经历可作为若干优秀品质 / 软能力的佐证。因此也建议按照项目经历的叙述方式，突出自己很厉害很优秀。和项目经历、实习经历相比，描述可以更精炼些。不要记流水账，不要只是写某年某月我做啥，要进一步挖掘和呈现出优秀品质 / 软能力。

校园经历要尽可能呈现在简历上。投编程等岗位的某些同学，觉得写这些有啥没啥的经历，没用！其实不然！专业技能符合岗位要求，并且优秀品质 / 软能力也符合岗位要求，则获得 offer 的几率才比较大。用人单位很看重优秀品质 / 软能力的。


自我评价

❇️ 可以呈现：
▫️ 3~4点，每点描述 1 个软能力，或几个相关软能力。
▫️ 每点不超过 1 行为宜。每行前面加数字或点，方便阅读。
▫️ 软能力大多为：沟通、协作、逻辑、抗压、……。
▫️ 可以简要呈现（重复）相关经历，以佐证的确具备某软能力。

🆖 建议不写：
▫️ 遵纪守法。这是必须满足的底线，不必重复。
▫️ 技术能力。这里主要是软能力。
▫️ 吹拉弹唱。除非作为某种软能力的佐证。
▫️ 运动健身。除非作为某种软能力的佐证。
