学投相辅 -- 24届校友L

来源：24届校友L | 更新：24-10-14 | 发布：24-10-14

2024年06月采访L校友。L校友最终入职心仪的国内著名人工智能上市公司，做Java开发岗位。L校友找工作的经历是有参考价值的，列出来供参考。L校友并不拔尖，TA最终能够找到心仪单位的心仪岗位，主要得益于：（1）勇于尝试不断投简历，从找工作小白到达人；（2）坚持投简历不放弃，石沉大海还是继续投；（3）同步增强技术等能力，一边补一边投。以下是主要内容，供感兴趣者参考。

1、为什么找工作而不是继续升学

感觉有些课程（比如人工智能、深度学习等）一窍不通且预期学不会，想学会预期会非常辛苦，所以决定找工作不再升学。

2、什么时候开始找工作的

从大三下至大四上启动找工作事宜，大致在秋招前。

3、刚开始找工作是怎么开展的

没有竞赛奖项，没有这个那个荣誉，基本是一穷二白。就写个简历，然后参考辅导员老师在就业QQ群中的招聘信息，开始投简历。现在回头看刚开始写的简历是惨不忍睹的，因此漫无目的投简历必定是石沉大海的。

4、后来是怎么改善的

盲投应该无效果，所以下定决心先学习。我喜欢Java开发，但水平不高，因此找B站课程学习提升技能。前后花费了2-3个月，进行学习和做教程中的项目。内容比较多，期间放弃2次各休息几天，差点没有坚持下来。做教程中的2个项目，按照教程敲了一遍代码。

在做完教程中第1个项目时，看到辅导员老师发的某软件公司GT的招聘信息，就投简历，很快收到该公司的面试邀请。在面试前还把大学主要课程温习一遍，为第一次面试做好充分准备。前期努力终于有回报，面试通过并很快得到offer，在放寒假之前。

5、获得第1个offer后还继续投简历吗

被某软件公司GT录用，使我找工作信心倍增。还是在QQ招聘群中看到某中国500强企业，觉得可以试试，就继续投简历。面试等都顺利通过，但迟迟不发offer。多次跟催后答复说人已招满。一边郁闷，一边赶紧又投了很多大企业的相关岗位。

[编者注]
- 后续这个中国500强某企业，也给L发了offer，原因是录取的同学不来了，有岗位空出来。
- 同学们投简历找工作时要保持好心态，不要被只读不回、婉拒等影响。
- 万千岗位，我只要一个，肯定能找到。
- 在一段时间、相关现状下，争取切合实际的最好的岗位。
- 不被手握 10 个offer的所影响，牛娃也不能干10个岗位。
- 和心仪公司不断联，能抓住空缺岗位。

6、怎么认为是大企业呢

从牛客网、智联招聘等可以看到，应该是不错的企业。大多没有回应，或者做完面试后没有进一步消息。最后进入心仪的某人工智能上市公司，招聘信息也是从QQ群看到的。

7、最终是如何进入心仪的某人工智能上市公司的

我很喜欢这家公司，投的还是Java开发岗。笔试和技术面试都顺利通过，接下来是最后2轮面试（HR面试和主管的综合面试）。为保证能顺利通过面试，我咨询有企业工作经验的老师关于最后2轮面试的经验，自己也从网上学习经验并准备可能的问题，比如怎么自我介绍、自己的优缺点等。

在综合面试时发现面试官表情严肃，我就模仿某大公司招聘秘书的个人介绍（比作鸳鸯火锅），我把自己比作麻婆豆腐，红色代表我对Java开发的热爱，刀工精细代表我对待技术严谨，一下子把面试气氛缓和了。

可能最终打动综面面试官的是我对于“从高中到现在，你人生最黑暗的经历是什么”的回答。我正好准备了缺点相关的回答可用上，说到我高中比较内向等等，大学里积极参加社团等活动改变自己。面试官认为是很好的校园经历但没有在简历上看到（我当初认为投的是Java开发技术岗，就没有描述这些可能加分的校园经历）。由于是真实的经历，在陈述过程中被自己都打动了，并成为展示相关软能力的好机会。

8、对于找工作最深刻的感想是什么

还是应该敢于投简历，其实要求没那么高，不要被招聘要求吓退。用人单位招聘的是本科应届毕业生，开出相应的薪资待遇，能力要求也是针对本科应届毕业生的，其实要求没那么高。

我被某人工智能上市公司录用后，推荐同学也投递并还提供内推码，但该同学最终未投递，觉得自己不行。其实该同学能力很强，比我厉害。

即使投简历、笔试面试、没录用，也没有损失，反而能增长找工作的经验。如同买卖一样，用人单位肯定希望用有竞争力的薪资待遇，买到“最好”的人才，因此一般招聘要求都略高。假定真的有用人单位用本科应届毕业生的薪酬待遇，招聘能力要求如同工作多年的技术专家，那这个单位不进也罢。

面试的时候问的技术问题，也没有那么深奥。

9、大学四年所学有用吗

非常有用！最重要的是学到如何学习的方法，其次是学到很多知识点。掌握如何学习的方法，在今后学习新东西时就知道如何入手，比如我为了找工作猛学Java就相对容易上手。

很多知识点在被用到时发现很有用，比如学过操作系统课程讲到的进程线程知识后，在学习Java的多线程编程就相对好理解。再比如计算机网络课程中讲解到很多网络协议知识，虽然在Java中使用时都是调用封装好的，但具备网络知识更容易理解。

来源：25届本科L | 更新：24-09-26 | 发布：24-09-26

题目1 海盗分金币

有5个海盗抢得100枚金币，他们按抽签的顺序依次提方案：
首先由1号提出分配方案，然后5人表决，投票要超过半数同意方案才被通过，否则他将被扔入大海喂鲨鱼，由下一位再次提出方案，依此类推。请问：1号最多能给自己分配并得到多少金币？

题目2 分桃子狼人杀

有四个同学，每个人得到的桃子数量不同，4个到7个之间。然后，4个人都吃掉了1个或2个桃，结果每个人剩下的桃数是还是各不相同。
4个人吃过桃后，说了如下的话。其中，吃了2个桃的人撒谎了，吃了1个桃的人说了实话。
安娜:“我吃过红色的桃。”
杰瑞:"安娜现在手里有4个桃。”
凯特:“我和简一共吃了 3个桃。”
简:"杰瑞吃了2个桃。凯特现在拿着的桃数量不是3个。”
请问:最初杰瑞手里有几个桃?
A 6
B 4
C 5
D 7

题目3 幼儿园问题

老师让幼儿园的小朋友排成一行,然后开始发水果.老师分发水果的方法是这样的：从左面第一个人开始,每隔2人发一个梨；从右边第一个人开始,每隔4人发一个苹果.如果分发后的结果有10个小朋友既得到了梨,又得到了苹果,那么这个幼儿园有多少个小朋友?

题目4 二叉搜索树的路径

假设某棵二叉查找树的所有键均为1到10的整数，现在我们要查找5，可能的路径为……

可能的思路：左子树的值都小于当前节点，右子树的值都大于当前节点

编程题：最长回文子序列

给你一个字符串，你可以删除任意字符，输出一个最长的回文子序列

思路：1.保证回文：反转原字符串，子字符串一致而是回文 2.删除（子序列）：求其公共子序列 

示例代码，表示下思路，不一定正确，不一定能运行哦
#include<iostream>
#include<string>
using namespace std;
int main(){
    string s;
    cin>>s;
    string res;
    for(int i = s.length()-1;i>=0;i--){
    res += s[i];
    }
    const int N = s.length();
    int dp[N+1][N+1];
    for(int i = 0;i<=N;i++){ 
        for(int j=0;j<=N;j++){ 
            dp[i][j]=0; 
        } 
    } 
    for(int i=0;i<=N-1;i++){ 
        for(int j=0;j<=N-1;j++){ 
            if(res[j]==s[i]){ 
                dp[i+1][j+1]=dp[i][j]+1; 
            }else{
                dp[i+1][j+1]=max(dp[i][j+1],dp[i+1][j]); 
            } 
        } 
    } 
    cout<<dp[N][N]<<endl; 
}

编程题K个三次方之和

题目描述： 给你一个整数N，要求a1^3+a2^3+.....+ak^3=N
a1 a2 可重复，要求K最小，输出一个长度为K的数组。
样例：
输入：42
输出：[2,2,2,2,2,1,1]
样例解释
这里并不能去每次找最大的三次幂让他相减，需要DP最短的路径，如果按照贪心的思想就会出现这样的错误42 [3,2,1,1,1,1,1,1,1]

积极投递不断尝试 -- 24届校友W

来源：24届校友W | 更新：24-09-16 | 发布：24-09-16

于2024年06月采访W校友。该校友最终入职某上市公司，从事销售岗位。该公司提供集成电路测试的服务。W校友在同学中也不是很突出，最终找到销售岗位也是 积极投递不断尝试 的结果。W校友结合找工作过程，有如下几点感想希望和学弟学妹们分享：

1、积极参加访企拓岗活动，到企业现场看看。 作为学生是不大清楚企业长什么样子的，因此找工作时自然而然会迷茫和困惑。我参加过某次访企拓岗活动后，到某些企业实地看了看，就大致感受到企业是什么样子，找工作更加有动力和方向了。

2、建议学院夯实就业指导课。 学校的就业指导课，对于要找工作的同学，还是很有帮助的。建议学院要夯实就业指导课的内容以及练习，更加符合实际情况，更加偏实战些。 （编者注：同学们应上好就业指导课，多多互动）

3、持续加强在招聘群发信息的工作。 我最终入职的单位的招聘信息，也是来自招聘群。学院要保持并持续加强该项工作，对找工作同学很有帮助。并建议采用“有个xx机会，适合不想编代码的同学，可以投递试试“等类似说法。

W校友还整理出指导文档，供学弟学妹们参考。如同W校友在文档中提及的：请您理解，我的分享仅供参考，因为每个人的经历和处境都有所不同。摘要如下：

关于求职就业的一些经验

一、声明
亲爱的求职者朋友们，您好！我在此想与大家分享一些我个人的求职经历和心得体会，内容于2024年6月25日编写，希望能够对您的职业道路有所启发。请您理解，我的分享仅供参考，因为每个人的经历和处境都有所不同。在求职旅程中，我遭遇了形形色色的挑战，也在挑战中成长。以下是我所获得的一些宝贵经验。

二、个人信息
在校成绩：GPA不满3；英语水平：顺利通过四六级英语考试；奖项：参加过一些比赛，不过没能拿奖，这部分也就不加赘述；就职公司：某上市公司，从事销售岗位。 （编者注：其他信息省略）

三、经验及建议

3.1 转变求职心态
对计算机专业的本科生来说，你在校期间接受的培养计划让你学习的课程大多宽且泛，虽然其中不乏一些很重要的计算机课程，但还是不够，计算机这个专业，更多的是需要你自己去学，如果你早早的就确定了自己未来的发展方向，并在这个方向的学习路线上不断推进，那么恭喜你，你相较于别人有了一个较为良好的求职基础。即使你现在还感到迷茫，也不必灰心。每个人的经历都是独一无二的，我们要做的不是等到求职的时候发现自己无甚优势而感到自怨自艾、日渐消沉，关键是要利用现有的资源去尝试、进步和弥补不足。

3.2 制作简历
很多人可能觉得自己在本科期间学了很多东西，但是学的都不够深入，使得你在制作自己的简历的时候不知道往简历上写什么，然后你就害怕，害怕自己的简历会显得空洞，害怕被老师或者面试官批斗，害怕失败，害怕受挫，如果你一直害怕的话而不去解决让你感到害怕的事，那么你就一直无法往前走，记住，你遇到的每一次挫折和困难，都是你在进步路上的关卡，跨过去了就表示你在进步。
如果可能，找到你的兴趣方向，制定学习计划，去学习，然后根据这个方向制作简历。在简历中，突出你的技能和项目经验，以及实习经历和校园经历。在书写这些内容的时候大体奉行“简洁而不贫瘠，详尽而不冗余”的原则，使用简洁大方的模板，如“职徒简历”网站提供的模板，可以让HR一眼看到你的母校 XX 大学。 （编者注：此处省略简历的截图）

3.2.1 选择个人照片：
选择一张较为正式但不死板的照片，可以适当进行ps一下，让你处于一个比较明丽的风格中，展现出青春活力。第一印象很重要，所以照片需要给人留下正面积极的印象。

3.2.2 描述专业技能：
在简历的专业技能部分，不仅要列出你掌握的技能，还要展开描述，让你的专业技能看起来不是那么空泛，例如：“熟练掌握C语言和Java语言，对Java面向对象编程的概念（如封装、继承和多态）有深入理解”，这样书写总比直接写“掌握C语言和Java语言”要好吧？ （编者注：不一定成立，仅供参考）

3.2.3 书写项目经验：
在项目经验部分，写几个你比较熟悉的项目，如果没有的话，挤出点时间去跟一两个项目，或者去熟悉几个项目也行，写在简历上的时候，简洁地介绍每个项目的背景（这个项目是什么？）、你的职责（你在这个项目里用什么技术实现了什么功能？）和项目成果（最终做出来的项目成果，或者你在这个项目中的收获）。使用粗体字突出关键技术，确保HR能够快速把握你的技术能力和项目贡献。当然，你要确保你写在简历上的东西他要是问起来，你要能说的出东西来。 （编者注：类似于 STAR 法则）

3.2.4 展示证书和奖项：
如果有相关证书和奖项，也应该包含在简历中。遵循原则：简洁而不贫瘠，详尽而不冗余。

3.2.5 讲述实习经历：
如果有实习经历的话一定要写在简历上，现阶段各企业招聘更倾向于招一些拥有实际工作经验的人，所以有实习经历的话一定要写上去。表述上也是按“在什么时间范畴内做了什么事，有什么成果”去叙述，同样遵循“简洁而不贫瘠，详尽而不冗余”的原则。 （编者注：类似于 STAR 法则）

3.2.6 校园经历的描述：（此处省略若干文字） （编者注：1、类似于 STAR 法则，不要记流水账； 2、一切有利于加分的都可以写上，同时控制版面占用不要过多。）

3.2.7 针对性简历的制作：
根据你申请的不同岗位，制作多份针对性强的简历。例如，如果申请技术岗位，那么简历应侧重于专业技能、项目经验和实习经历。如果是申请银行或国企等岗位，那么应该更全面地展示你的专业性、校园经历和实习经历。 （编者注：同学自行在投递过程中证实）

3.3 求职准备
如果你对简历制作感到困惑，可以先制作一份初稿，然后寻求反馈和建议，同学、老师、网络等都是你能寻求到帮助的途径。不要怕被骂，做好心理准备，我最初的简历也是不堪入目，我去找了一位简历咨询处的老师时，被他狠狠批斗了，不过这个过程也学到了不少东西。当你有了一份简历之后，你就可以尝试着去投递各种岗位了，大多都是要网申，所以最好提前备好一些需要的材料，不然你会在这个网申的过程中感觉很难受，具体的可以参考图片中的内容，除此之外，如果还有其他技能证书或者奖项证书，也可以提前准备好，这会让你在网申的过程中顺利很多，毕竟这个过程十分繁琐，你投的是几十甚至上百家公司，所以提前准备好材料会方便一些。（原文此处是截图，包括了：CET6（证书？），彩色证件照，成绩单，成绩排名，教育部学籍在线验证报告，身份证，在校证明，最高学历学生证，……）

3.4 投递简历
至于往哪投递，学校会有很多场校园招聘会，不要错过，都去转一转，看一看，或许就会有意想不到的收获，当然还有各个国企、央企、各行业龙头企业每年针对应届生的秋招和春招，这都是可以在他们对应的官网上申请的。

3.5准备笔试
过了简历关就是笔试关了，计算机投的岗位基本都是笔试，如果投的是开发研发岗，那么笔试的内容基本就是一些算法题，一些专业题；如果是其他的一些岗位，可能就是一些逻辑思维题、数学题、英语题，还会有性格测试，这部分内容你可以针对性的去做准备。 （编者注：同学自行在投递过程中证实）

3.6准备面试
等到了面试关，由于你投递的岗位不一样，所以问你的问题也会不一样，不过大致也是根据你简历上的内容来进行提问，不管是开发岗还是其他岗，基本都会要求你进行一个简短的自我介绍，这个需要你提前准备，

  • 讲一下你的身份。例如，尊敬的面试官，您好！我叫***，是 xx 大学 xx 专业的应届毕业生。
  • 讲一下你在专业学习方面的一些经历。例如，在专业学习方面，我掌握了C，C++，Java等编程语言，熟悉了计算机组成，计算机网络，数据结构，操作系统，数据库等基础理论，参与了多个课程项目。除此之外，个人兴趣驱使我自学了Java后端开发技术，并有与他人合作共同开发项目的经历。我还通过了英语四级和六级考试
  • 讲一下你的课外的一些经历。例如，除了专业学习外，我还积极参与社团活动和社会实践，担任过 xx，组织策划了多次团课活动和班级团建活动；也担任过xx 志愿者。我还利用课余时间参加了一些兼职工作（和实习工作、家教、信息维护等）
  • 最后总结一下。我认为自己是一个积极进取、勤奋好学、责任心强的人。我对***的发展前景和企业文化非常感兴趣，希望能够成为***的一员，为***的发展贡献自己的力量。

还有其他的一些问题。开发岗可能问一下 面试八股文，这个多去背一背就行了，其他的可能还有比如 “你的优势和劣势”、“为什么选择我们公司”、“为什么选择这个岗位” 等问题，当如可能还会有其他的很多问题，这个需要你在网上去查询相关的信息，例如像牛客之类的网站，会有人分享在某个公司某个岗位的面试经历（也称面经），求职过程中也可以多看看这些内容，能让你对面试中会出现的问题有个预期，当然，你不能完全靠提前准备就能解决所有的问题，有时候需要你临时发挥，那么这部分就靠你自己了。

四、结语
以上分享的所有建议和心得，都源自我个人的经历和感悟。它们可能不尽完美，但我真诚地希望它们能为正在求职道路上的朋友们提供一些指导和帮助。在求职的旅程开始时，恐惧、担忧和迷茫是自然的情绪。然而，一旦您勇敢地迈出第一步，就会发现接下来的路逐渐变得清晰。不断地尝试、经历挑战、感受成长的过程，这些都将成为您宝贵的财富。我衷心希望每一位求职者都能够获得理想的offer，开启自己的职业生涯。
